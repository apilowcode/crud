
# 产品介绍

## 产品定义

基于 JSON 与 SQL 模板配置，自动化生成标准化 RESTful 接口，将 API 交付周期缩短至分钟级，让开发者从重复的 CRUD 中解放出来，专注核心业务逻辑。

## 开发背景

### 传统模式的痛点

在传统后端开发中，CRUD 接口占据了大量研发资源。即便引入代码生成器或 ORM 框架，开发者仍需维护庞大的实体类与映射代码，导致“代码膨胀”与维护成本高企。

### 现有方案的局限

虽然 APIJSON 等动态查询技术试图将部分逻辑前置以简化后端，但其往往牺牲了接口的规范性与安全性，导致接口定义失控、权限管理复杂，难以满足企业级应用对稳定性的要求。

### 我们的方案

基于 **APIJSON + MyBatis** 构建标准化接口实现层，通过**集中式配置管理**统一管控接口路由、操作类型、数据模板（JSON/SQL）及响应格式，并内置细粒度的访问控制机制。

## 产品特性

### 1. 极致效能：低代码驱动，告别 CRUD 冗余

声明式低代码开发模式，通过配置替代编码，显著降低开发门槛与复杂度。有效抑制后端接口层的“样板代码”膨胀，将研发资源从重复劳动中解放出来，聚焦核心业务逻辑创新。

### 2. 标准统一：构建企业级 API 治理体系

内置严格的 RESTful 规范引擎，自动统一 URI 路径设计、请求参数校验及响应数据格式。消除团队间的风格差异，确保系统间 API 的一致性与可维护性，为微服务架构奠定坚实基础。

### 3. 模板引擎：JSON + SQL 双核驱动

独创“APIJSON 结构定义 + MyBatis SQL 映射”的双模开发机制。开发者只需编写轻量级的 JSON 配置与 SQL 模板，即可动态生成复杂接口。既保留 SQL 的强大查询能力，又享受 JSON 的结构化便利，实现数据操作逻辑的极简表达。

### 4. 广泛兼容：多源异构数据库无缝接入

社区版原生支持 MySQL、PostgreSQL、Oracle、SQL Server、SQLite，企业版在社区版的基础上新增金仓、达梦、神州通用、高斯、海量等国产关系数据库支持。具备强大的多数据源管理能力，支持在同一应用中混合连接多个异构数据库，轻松应对遗留系统整合与跨库查询场景。

## 核心价值

### 1. 极致的开发效率

| 对比维度 | 传统模式                                                                                | 配置模式                                                  |
| -------- | --------------------------------------------------------------------------------------- | --------------------------------------------------------- |
| 开发流程 | 建表 → 写 Entity → 写 Mapper XML → 写 Service → 写 Controller → 调试 Swagger → 前端联调 | 配置 SQL 或 JSON → 自动生成 API 文档和接口 → 前端直接调用 |
| 耗时     | **半天 ~ 一天**                                                                         | **几分钟**                                                |
| 适用场景 | 复杂业务逻辑                                                                            | 内部管理后台、临时数据查询、标准增删改查                  |

### 2. 降低后端门槛

中小型公司或非技术主导的项目往往缺乏资深 Java 后端工程师。通过可视化配置 SQL 和 JSON，初级工程师甚至产品经理即可完成基础接口搭建，让高级后端工程师专注于复杂业务逻辑。

### 3. 标准化与治理

强制统一返回格式、异常处理、权限拦截，避免不同开发人员写出风格迥异的代码，大幅降低后期维护和交接成本。

### 4. 动态热加载

支持**热加载**配置，修改 SQL 或 JSON 后无需重新编译部署即可生效新接口，在需求频繁变更的场景中极具优势。
